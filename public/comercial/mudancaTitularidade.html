Mudança de Titularidade

Processo utilizado quando um cliente deseja transferir a responsabilidade do contrato para outra pessoa.

Objetivo Transferir contrato, login e responsabilidade para outro titular
Ponto crítico Verificação financeira e histórico do novo titular
Regra Nunca prosseguir com pendência financeira

1. Validações obrigatórias

Regra crítica: A mudança de titularidade só pode acontecer após validação completa de ambos os lados.
TITULAR ATUAL
  • Verificar débitos em aberto
  • Confirmar situação do contrato
NOVO TITULAR
  • Verificar se já é cliente
  • Consultar histórico
  • Verificar débitos antigos
  • Verificar equipamentos não devolvidos

2. Validação com o financeiro

  1. Consultar débitos do titular atual
  2. Se houver débito → não prosseguir
  3. Consultar situação do novo titular
  4. Verificar pendências financeiras antigas
Bloqueio obrigatório: Se existir débito ou equipamento não devolvido, o processo só continua após regularização.
Perfeito, vou verificar aqui a situação do contrato e já retorno para dar continuidade com segurança.

3. Análise do novo titular

Passos

  1. Buscar o novo titular no IXC
  2. Se existir cadastro → validar histórico
  3. Se não existir → seguir como novo cliente
Mesmo que o novo titular já exista no sistema, ele deve passar pela mesma validação financeira.

4. Processo no Sales

Após validação, o processo segue igual ao Novo Contrato.

  1. Criar lead
  2. Preencher dados
  3. Fazer viabilidade
  4. Confirmar endereço
  5. Gerar negociação

5. Vencemos Wizard

Ponto crítico: Atendimento correto → ID 50 - MUDANÇA DE TITULARIDADE
  • Contrato: novo contrato
  • Produto: plano escolhido
  • Acesso: conforme padrão
  • Atendimento: ID 50

6. NOC e Estoque

NOC

  • Solicitar transferência do login
  • Do titular antigo → novo titular

Estoque

  • Solicitar transferência do comodato
  • Garantir vínculo correto do equipamento
Erro crítico: Não transferir login ou comodato corretamente pode gerar inconsistência grave no sistema.

7. Finalização

  1. Gerar contrato
  2. Enviar assinatura
  3. Agendar (se necessário)
  4. Confirmar ativação
  5. Cancelar contrato antigo
  6. Inativar titular antigo
Obrigatório: O contrato antigo deve ser cancelado após conclusão.

8. Erros que não podem acontecer

  • Não validar débitos
  • Ignorar histórico do novo titular
  • Não transferir login
  • Não transferir comodato
  • Manter contrato antigo ativo
  • Executar processo com pendências financeiras

9. Resumo rápido

  1. Cliente solicita mudança
  2. Validar financeiro (ambos)
  3. Validar novo titular
  4. Criar lead
  5. Fazer viabilidade
  6. Vencer Wizard (ID 50)
  7. Transferir login (NOC)
  8. Transferir comodato (estoque)
  9. Gerar contrato e assinatura
  10. Cancelar contrato antigo
